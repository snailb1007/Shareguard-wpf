<task type="auto">
  <name>Task 4: Implement WindowsClipboardMonitorService</name>
  <files>Shareguard-wpf/Services/IClipboardMonitorService.cs, Shareguard-wpf/Services/WindowsClipboardMonitorService.cs</files>
  <read_first>Shareguard-wpf/App.xaml.cs</read_first>
  <action>
    - Create IClipboardMonitorService.cs in Shareguard-wpf/Services/ defining:
      - Event `UrlCleaned(string before, string after, int removedCount)`
      - Methods: `StartMonitoring(IntPtr hwndSource)`, `StopMonitoring()`, and property `IsMonitoring`.
    - Create WindowsClipboardMonitorService.cs in Shareguard-wpf/Services/ implementing IClipboardMonitorService:
      1. Hook into window message loop using `HwndSource.AddHook`.
      2. Call Win32 `AddClipboardFormatListener(hwnd)` on start and `RemoveClipboardFormatListener(hwnd)` on stop/dispose.
      3. Listen for the `WM_CLIPBOARDUPDATE` (0x031D) message.
      4. On message receipt, read clipboard text. If it is a valid HTTP/HTTPS URL, call `IUrlCleanerService.CleanUrl`.
      5. If cleaned, write it back to the clipboard. Avoid self-triggered write loops by ensuring we do not re-clean already-cleaned text (or transiently bypass next update hook).
      6. Raise the `UrlCleaned` event on successful clean.
  </action>
  <verify>dotnet build Shareguard-wpf succeeds</verify>

</task>
