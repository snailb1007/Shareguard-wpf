## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline helper text offsets, compact menu separators |
| sm | 8px | Compact control gaps, tray menu grouping rhythm, toast text gaps |
| md | 16px | Default element spacing, settings row padding, card padding |
| lg | 24px | Window edge padding and main layout gutters |
| xl | 32px | Settings section breaks and major panel separation |
| 2xl | 48px | Empty-state vertical breathing room |
| 3xl | 64px | Page-level spacing only; avoid inside compact settings/tray surfaces |

Exceptions: icon-only buttons and toggle hit targets must be at least 44px tall/wide even when visual glyphs are smaller; native tray context menu sizing follows Windows shell metrics.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #0F172A | Main window background and persistent app surfaces |
| Secondary (30%) | #1E293B | Settings panels, tray-status panel content, cards, list backgrounds |
| Accent (10%) | #3B82F6 | Primary action buttons, active settings toggles, selected tab/section indicators, hotkey capture focus ring |
| Destructive | #EF4444 | Exit warnings, hotkey registration failures, destructive confirmation text/actions |

Accent reserved for: `Clean Clipboard Now`, active clipboard monitoring toggle, selected settings/navigation section, progress bars, focused hotkey capture input, and successful primary restore/open affordances. Do not use accent for every clickable control; secondary actions remain slate (`#1E293B` / `#334155`).

Semantic colors:
- Success: #22C55E for "clipboard cleaned", "hotkey registered", and clean-copy success status only.
- Warning: #F59E0B for recoverable conflicts such as "hotkey already in use" or "custom directory unavailable; using original folder".
- Muted text: #94A3B8 for descriptions; #64748B for low-priority metadata and status-bar text.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | Clean Clipboard Now |
| Empty state heading | Desktop shortcuts are ready |
| Empty state body | Use the tray menu or Ctrl+Shift+G to clean the clipboard without opening ShareGuard. |
| Error state | Desktop shortcut could not start. Choose a different hotkey or use the tray menu. |
| Destructive confirmation | Exit ShareGuard: ShareGuard will stop monitoring the clipboard and the global hotkey will be disabled. Exit? |

Additional required labels:
- Tray tooltip: `ShareGuard - privacy tools running locally`
- Tray default action: `Open ShareGuard`
- Tray menu items, in order: `Open ShareGuard`, `Clean Clipboard Now`, `Pause Clipboard Monitor` or `Resume Clipboard Monitor`, `Settings`, separator, `Exit ShareGuard`
- Hotkey label: `Global hotkey`
- Hotkey default value: `Ctrl+Shift+G`
- Hotkey conflict copy: `Ctrl+Shift+G is already in use. Choose another shortcut.`
- Clipboard monitoring toggle label: `Monitor clipboard`
- Clipboard monitoring helper: `Automatically cleans copied URLs. You can turn this off anytime.`
- Clipboard monitoring tray labels: `Pause Clipboard Monitor` when active; `Resume Clipboard Monitor` when paused.
- Notification toggle label: `Show clean notifications`
- Output directory setting label: `Clean copy location`
- Output directory options: `Same folder as original`, `Choose folder...`
- Output naming label: `Clean copy naming`
- Output naming preview: `Example: photo.clean.jpg`

## Layout & Interaction Details

Source: REQUIREMENTS.md REQ-05, ROADMAP Phase 5 success criteria, Phase 1/2 deferred decisions, Phase 5 RESEARCH.md, product design desktop entry points.

1. **Tray Residency Contract**
   - Closing the main window hides it to the tray; it must not quit the app.
   - The tray icon persists independently of the main window lifecycle.
   - Double-clicking or left-clicking the tray icon restores the main window, sets `WindowState=Normal`, brings it to foreground, and focuses it.
   - Right-click opens a native tray context menu with the exact copy and order from the copywriting contract.
   - `Exit ShareGuard` is the only tray action that ends the process; show the destructive confirmation first.
   - Dispose the tray icon during application exit so no stale tray icon remains.

2. **Tray Visual Contract**
   - Use a high-resolution `.ico` with 16x16, 32x32, and 48x48 frames.
   - Tooltip text must use the tray tooltip copy exactly.
   - Tray menu command labels use native Windows menu styling; do not build a custom acrylic/floating menu for the tray.
   - Use separators only between grouped actions: open/clean/settings and exit.

3. **Visual Hierarchy Contract**
   - The primary focal point in the main window remains the existing clean workflow, with `Clean Clipboard Now` as the strongest action.
   - Desktop convenience controls are secondary utility settings; they must appear as compact rows and must not compete visually with file cleaning/history content.
   - Within the settings surface, visual priority is: clipboard monitoring state, global hotkey readiness/conflict, clean copy location, notification preference.
   - Use the accent color only for the active state, focused hotkey capture field, primary clean command, and selected settings/navigation indicator.

4. **Hotkey Interaction Contract**
   - Default hotkey is `Ctrl+Shift+G`.
   - The hotkey action is `Clean Clipboard Now`: read current clipboard once, clean if applicable, update clipboard, show status/notification, and log history.
   - If the window is hidden, a successful hotkey clean may keep the window hidden and show only the existing notification/status path.
   - If the hotkey is registered successfully, show success status text: `Ctrl+Shift+G is ready.`
   - If registration fails, show the hotkey conflict copy inline in settings using warning/destructive color; do not crash, block startup, or show repeated modal dialogs.
   - A hotkey capture control must have a 44px minimum height and an accent focus ring. Changing a hotkey at runtime must unregister the old shortcut before registering the new one.

5. **Settings Surface Contract**
   - Add a compact `Settings` tab or section inside the existing main window; do not create a separate landing page or modal-only settings flow.
   - Use full-width settings rows, not nested cards. Each row has a heading label, one-line helper text, and the control aligned right.
   - Controls required in this phase: `Monitor clipboard` toggle, `Show clean notifications` toggle, `Global hotkey` capture, `Clean copy location` selector, `Clean copy naming` preview.
   - Settings changes apply immediately and update the status bar with concise feedback.
   - Persisted settings should be represented visually on startup before background services begin acting on them.

6. **Directory Customization Contract**
   - Default remains `Same folder as original`, preserving Phase 1 behavior.
   - `Choose folder...` opens the standard Windows folder picker and displays the selected path with middle ellipsis trimming and full path in tooltip.
   - If the chosen folder is unavailable, keep the setting visible, show warning copy, and fall back to same-folder output for that operation only.
   - Never offer replace-original behavior in this phase.

7. **Notification Contract**
   - Reuse the existing notification style: dark translucent surface, shield/check icon, success green for cleaned events, compact two-line text.
   - Hotkey success title: `Clipboard Cleaned`
   - Hotkey success body: `Removed {count} tracking parameter(s).`
   - No-change body: `No tracking parameters found.`
   - Notifications must not steal focus.

8. **Responsive/Window Contract**
   - Maintain the current 1000x650 desktop-first window sizing.
   - Settings rows must remain readable at 900px width; long paths and hotkey labels must trim, not overflow.
   - Avoid new hero areas, oversized illustrations, or decorative backgrounds; this phase is a utility/control surface.
